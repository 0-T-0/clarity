Highlights

  • Updated to TypeScript 2.0 and Angular 2.2.0

    With the 0.7.1 release, Clarity has been updated to the latest versions of Angular and Typescript.

  • Documentation improvements on website

    In response to requests from the community, the Clarity team updated the layout and design of the documentation section of the website. We're hoping these improvements make our research and code examples easier to read and navigate.

Bug Fixes

  • Fixed a font weight issue with text links in Clarity headers
  • Fixed an issue with the alignment of arrows in stackview labels that was introduced after the move away from icon fonts
  • Fixed tests that were failing because of replace anchor tags in tabs/nav with buttons
  • Fixed an issue with the nav/tab highlight not overlapping the border below the tabs
  • Removed extra padding around progress blocks in cards after cards refactor in 0.6.2
  • Fixed test in dropdowns that was returning a false positive
  • Hid the branding icon on tablet-sized and smaller screens
  • Fixed an issue with alignment and cursors on responsive nav triggers
  • Fixed a vertical alignment issue in small alerts
  • Fixed a peer dependency issue in Clarity Icons that was linking directly to Custom Elements' GitHub repo instead of npm

Other Changes

  • Added copyright information to all files in the repository
  • Updated the website and README with instructions on using Clarity with angular-cli
  • Various website and documentation updates
